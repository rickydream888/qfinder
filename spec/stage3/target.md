# 目标

为 qfinder 的预览面板增加对 4 类常见办公文档的「轻量首屏预览」能力：

| 类型 | 预览内容 |
| --- | --- |
| `.xlsx` | 第 1 个工作表的前 100 行 × 前 20 列 |
| `.pdf`  | 第 1 页 |
| `.docx` | 文档开头若干段（按段落数 / 字符数截断） |
| `.pptx` | 第 1 张幻灯片（依赖系统已安装 LibreOffice，转 PDF 后取首页） |

设计原则：

1. **限制大文件资源占用**：所有格式均设上限（文件大小 + 内容截断），坏文件 / 超大文件不得拖垮 UI 与内存。
2. **无强制外部依赖**：xlsx / pdf / docx 必须在「未安装任何第三方系统软件」的机器上工作；pptx 缺 LibreOffice 时优雅降级到提示信息。
3. **复用现有架构**：仍走 `preview` Tauri 命令；新增渲染分支以 `PreviewPayload` 的 `kind` 区分，前端按 kind 选择渲染器。
4. **离线可用**：所有第三方 JS 库（PDF.js、mammoth.js）随包发布，不从 CDN 加载。
5. **不破坏 Stage 1 / Stage 2**：原有目录、文本、图片预览以及 Material 图标方案保持不变。
