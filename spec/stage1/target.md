# 目标

开发一个基于 `Rust` + `Tauri2` 的跨平台文件管理和预览的软件工具。

## 功能

本软件具备两种模式：

- 左右各一列文件系统树的管理模式；
- 左侧为文件系统树，右侧为文件预览区的预览模式。

### 文件系统树

最上方有一个`checkbox` + “显示隐藏目录和文件”的文本标签，用于选择是否展示隐藏目录和文件，默认不选中。

下面就是文件系统树，在不同平台下有不同的根部元素：

**Windows**

- 用户的家目录；
- 所有驱动器（含网络驱动器）。

**macOS**

- 用户的家目录；
- 系统根目录（一般是Macintosh HD）；
- iCloud目录（如果有的话）；
- 挂载到/Volumes目录下的除了系统磁盘以外的目录）。

**Linux**

- 用户的家目录；
- 系统根目录；
- 挂载的光盘和其他介质。

文件系统树的功能：

- 点击目录前的符号展开/收起目录的内容，双击目录展开/收起目录的内容，每次展开时重新加载目录的内容；
- 展开是向下的层级式展开（和Windows资源管理器左边窗格的行为类似）而不是向右展开一大堆目录和文件；
- 使用鼠标拖动目录或文件进行移动操作；
- 在Windows和Linux平台下：Ctrl+C（复制）、Ctrl+X（剪切）、Ctrl+V（粘贴）、Delete（删除）；
- 在macaOS平台下：Cmd+C（复制）、Cmd+X（剪切）、Cmd+V（粘贴）、Delete（删除）；
- 在选中目录或文件的情况下再单击它，进入inline的重命名模式，注意和双击区别；
- 在有双列文件系统树的情况下，可以跨树移动、复制、剪切、粘贴；
- 删除均为移动到回收站/垃圾桶/废纸篓的可恢复的删除；
- 双击文件使用系统默认方式打开文件；
- 不要随意给文件系统树的区域加上背景，只能给选中的目录或文件加上背景。

### 预览

在预览模式下，用户选中文件系统中的目录时，在预览区显示：

- 该目录下有多少个子目录和多少个文件（不含子目录中的文件），在系统中有`du`命令时（一般是在macOS或Linux下）使用该命令计算选中目录的总磁盘用量，用人类友好的方式展示。

用户选中文件时，在预览区显示：

- 文件为纯文本类型时（例如.txt、.md、.html等，或可以使用`file`命令判断出它是个纯文本文件），在预览区显示文件内容（该换行的要换行），并以预览区大小为限制，超出的内容不显示；
- 预览纯文本文件时如果文件大小超过10KB，那么仅读取最前面的10KB内容进行预览；
- 文件为可显示的图片类型时，则根据预览区的大小显示缩放后适应的图片内容；
- 预览图片文件时如果文件大小超过20MB，那么显示文件大小并提示超过预览限制；
- 文件为其他类型时显示文件大小。

### 界面

**最上层**

- 左侧是模式切换按钮，默认为管理模式（即双列文件系统树模式），此时按钮上的文字为“切换到预览模式”，点击切换为预览模式（即文件系统树+预览区），此时按钮上的文字变为“切换到管理模式”；
- 右侧是后台任务展示区，同一时间只允许执行一个后台任务：即目录或文件的重命名、删除、复制或移动（剪切）操作。该区域需展示任务的目标（重命名xx到yy、删除xx、复制xx到yy或移动xx到yy）、任务执行时间（秒）和执行状态（进行中、已完成），存在未完成的任务时不允许执行新的任务，用户操作时要弹出对话框提示用户有后台任务正在运行，暂不支持添加新的任务，但此时用户仍可在文件系统树种展开、收起目录或对目录、文件进行预览。

**主体**

- 管理模式下包含左右双列文件系统树；
- 预览模式下包含左边的文件系统树和右边的预览区；
- 每个文件系统树有自己的隐藏目录和文件展示`checkbox`。

只需要开发Light Mode下的软件即可，无需支持Dark mode之类的样式。

用户进行非法操作或无操作权限时弹出对话框提醒用户并不进行相关的操作。

### 开发

相关开发资源：

- 在`resources/css`目录下已有`Bootstrap`的CSS文件；
- 在`resources/js`目录下已有`Bootstrap`和`jQuery`的JavaScript文件；
- 在`resources/icons`目录下已有`Bootstrap`提供的图标文件。

有需要的话复制相关文件到其他目录，不要直接引用`resources`目录下的内容，该目录不会提交到代码仓库。

Rust GUI框架请使用`Tauri2`而不是它的1.x版本，如果对`Tarui2`不太熟悉请先去阅读它的文档或代码。

不得使用node.js/npm/npx等相关的前端工具，页面由最基本的HTML + CSS + JavaScript技术栈开发，仅支持引入`Bootstrap`和`jQuery`。

### 注意点

- 先剪切再粘贴等效于移动目录或文件，不要先复制再删除；
- 注意网页和Rust之间的通信，确保能正确地相互调用；
- 拖动目录或文件时需要有拖动的画面效果，不要让鼠标光秃秃的移动；
- 双击展开/收起目录和先选中再单机进入重命名，这两个要区分好；
- Windows路径中的"\"在JavaScript代码中要处理号，不能直接放到字符串里，否则有可能会变成转义符号；
- 再次强调不得使用node.js/npm/npx等相关的前端工具。
